ماژول پیش‌پردازش متن (preprocessing.py)

log_function_call(func)

دکوراتور برای لاگ‌گیری ورودی و خروجی هر تابع. قبل و بعد از اجرای تابع، لاگ DEBUG می‌زند.

clean_word(word)

پاکسازی یک کلمه از کاراکترهای نادیده گرفته‌شده (SET_A - {' ', 'ـ'})

خروجی: (کلمه پاک‌شده, لیست کاراکترهای حذف‌شده)

تغییر روی متن: حذف کاراکترهای غیر ضروری.

get_removed_chars(word)

فقط لیست کاراکترهایی که حذف شده‌اند را برمی‌گرداند. متن اصلی تغییر نمی‌کند.

is_diacritic(char)

بررسی اینکه آیا کاراکتر یک دیاتریک (علامت نشانه‌گذاری صداها در فارسی/عربی) است یا نه.

ورودی: char → یک کاراکتر (مثلاً 'َ' یا 'ُ')
خروجی: True اگر کاراکتر دیاتریک باشد، False اگر کاراکتر معمولی یا غیر دیاتریک باشد.
is_phonetically_silent_vav(word, pos)

بررسی اینکه آیا حرف «و» در موقعیت مشخص بی‌صدا است یا نه. متن اصلی تغییر نمی‌کند. (مانند کلمه 'خواهر')

convert_to_phonetic(text)

تبدیل متن فارسی/عربی به نمایش فونتیک (مثلاً «و» → v یا u) (نگاشت فونتیک)

متن اصلی → متن فونتیک.

validate_text_length(view_func)

دکوراتور برای بررسی طول متن و تعداد کلمات. اگر طول یا تعداد کلمات بیش از حد باشد، خطا می‌دهد و اجرای تابع بعدی متوقف می‌شود.

ماژول هم‌ترازی و هایلایت (highlighting.py)

highlight_differences(word1, word2, phonetic_mode=False)

این تابع دو کلمه را کاراکتر به کاراکتر مقایسه می‌کند و تغییرات را به شکل هایلایت رنگی و عملیات قابل فهم ثبت می‌کند.

ورودی‌ها:
  • word1 → کلمه اول
  • word2 → کلمه دوم
  • phonetic_mode → اگر True باشد، مقایسه بر اساس فونتیک انجام می‌شود، نه حروف دقیق
خروجی‌ها:
  1. highlighted1 → کلمه اول با <span style="color:..."> برای نمایش تغییرات
  2. highlighted2 → کلمه دوم با رنگ برای تغییرات
  3. operations → لیست توضیحاتی مثل:
    • Match: 'ا'
    • Insert: 'و'
    • Delete: 'ر'
    • Substitute: 'ب' with 'پ' (Cost: 0.5)
مراحل عملکرد:
  1. ابتدا اگر یکی از کلمات خالی باشد، عملیات Insert/Delete انجام می‌شود.
  2. کلمات با clean_word پاک‌سازی می‌شوند (حذف کاراکترهای بی‌اهمیت).
  3. اگر حالت فونتیک فعال باشد، هر کلمه با convert_to_phonetic تبدیل به فرم فونتیکی می‌شود.
  4. الگوریتم Levenshtein با مسیر (path) اجرا می‌شود تا کمترین تغییرات بین دو کلمه پیدا شود.
  5. مسیر حاصل (match, substitute, insert, delete) تفسیر و رنگ‌دهی می‌شود:
    • match: بدون تغییر
    • substitute: رنگ متفاوت بسته به هزینه و نوع جایگزینی
    • delete: قرمز
    • insert: سبز
highlight_aligned_words(aligned_words1, aligned_words2, aligned_pairs, phonetic_mode=False)

این تابع برای لیست کلمات هم‌تراز شده در یک جمله یا متن عملیات مشابه highlight_differences را انجام می‌دهد، ولی به جای یک کلمه، روی کل لیست اعمال می‌شود.

ورودی‌ها:
  • aligned_words1 → کلمات متن اول (ممکن است _ برای جای خالی باشد)
  • aligned_words2 → کلمات متن دوم
  • aligned_pairs → لیست جفت شاخص‌ها (i, j) که کلمات متناظر را مشخص می‌کند
  • phonetic_mode → فعال‌سازی مقایسه فونتیکی
خروجی‌ها:
  1. highlighted_words1 → لیست کلمات اول با رنگ‌بندی
  2. highlighted_words2 → لیست کلمات دوم با رنگ‌بندی
  3. operations_texts → لیست کلی عملیات تمام کلمات

ماژول تولید گزارش (report.py)

generate_comparison_report(text1, text2, highlighted_text1, highlighted_text2, operations, total_cost, mode)

ایجاد یک دیکشنری کامل گزارش مقایسه متون.

شامل متن‌ها، هایلایت‌ها، عملیات، هزینه کل، حالت مقایسه و زمان تولید.

ورودی‌ها:
  • text1, text2 – متن‌های اصلی
  • highlighted_text1, highlighted_text2 – متن‌های هایلایت شده
  • operations – لیست عملیات تغییر
  • total_cost – مجموع هزینه تغییر
  • mode – حالت مقایسه
save_report_to_markdown(report, filename="comparison_report.md")

تولید یک فایل Markdown از گزارش، شامل: متن‌ها، هایلایت‌ها، جدول آمار عملیات، نمودار میله‌ای و جزئیات عملیات.

فایل را در پوشه reports ذخیره می‌کند.

download_report(report) (نسخه وب برای Django)

تولید فایل Markdown و بازگرداندن آن به صورت قابل دانلود برای کاربران وب.

روند اجرای کلی هنگام مقایسه دو متن

۱. validate_text_length

بررسی طول متن و تعداد کلمات. اجرای بقیه توابع متوقف می‌شود اگر متن طولانی باشد.

۲. پیش‌پردازش هر کلمه:
  • clean_word → حذف کاراکترهای غیرضروری
  • convert_to_phonetic → تبدیل به فونتیک (در صورت فعال بودن حالت phonetic)
  • is_phonetically_silent_vav → تحلیل حرف «و» بی‌صدا
۳. هم‌ترازی و تولید عملیات:
  • levenshtein_with_path → مسیر کمترین هزینه تغییرات
  • highlight_differences → تولید رنگ‌ها و عملیات برای هر کلمه
  • highlight_aligned_words → جمع‌آوری نتایج برای کل جمله
۴. محاسبه هزینه کل تغییرات

جمع کردن هزینه‌های insert, delete, substitute

۵. تولید گزارش نهایی:
  • generate_comparison_report → دیکشنری شامل متن، هایلایت، عملیات و هزینه کل
  • print_report → نمایش در کنسول

خلاصه تغییرات روی متن

مرحله تغییر روی متن
